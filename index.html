The Dot That Broke Everything

A debugging tale

🎬 The Mystery: My PM keeps complaining that creating employees fails, but it works perfectly for me. I create two employees. No problem. They try the same email... 💥 BOOM!

The culprit?

example.test@gmail.com

Remove the dot? Works like magic. ✨

❌ THE BUG The Innocent-Looking Code
// Auto-generate username from email const username = args.email .split('@')[0] .toLowerCase(); // Result: "example.test" 🤦‍♂️ // Clerk: "Username can only // contain letters, numbers and - or _"
➜
✅ THE FIX One Line. Pure Zen.
// Replace ALL invalid chars with _ const username = args.email .split('@')[0] .toLowerCase() .replace(/[^a-z0-9_-]/g, '_'); // 🎯 // Result: "example_test" ✨ // Clerk: "Welcome aboard!"

💡 What went wrong?

Clerk has TWO different fields:

  • Email → Allows dots (✓)
  • Username → NO dots allowed (✗)

I was auto-generating usernames from email prefixes WITHOUT sanitizing special characters!

🧠 Lesson learned: When debugging, always check what you're NOT seeing.
The error said "username" but I was thinking "email" 🤡

⏱️ 3 hours debugging. 1 character fix.

Classic developer life. 😅
